# Wrasul LOL Item Build Importer 工作规则

## 项目目标
把 Wrasul 在 B 站发布的英雄联盟出装 JSON，转换并安装到英雄联盟客户端支持的本地物品方案目录中。

## 安全边界
- 只读写英雄联盟安装目录下的 `Config\\Global\\Recommended` 或 `Config\\Champions\\<championKey>\\Recommended` 物品方案 JSON。
- 不读写游戏进程内存，不注入 DLL，不模拟按键鼠标，不拦截或改写网络请求。
- 不保存 Riot 账号、密码、登录态、令牌或任何个人隐私。
- 默认安装前创建备份；不允许静默覆盖用户原有配置。
- 脚本只在游戏和客户端关闭时运行，避免配置被客户端刷新覆盖。

## 文件结构约定
- `src/`：脚本源码。
- `tests/`：自动化测试。
- `examples/`：示例输入，只放可公开分享的样本或最小样本。
- `docs/`：流程说明、安全说明、调研笔记。
- `STATUS.md`：历史进度记录，不要求每次会话同步。

## 实施规则
- 默认中文文档，代码、命令、变量名使用英文。
- 普通用户入口优先：双击 EXE -> 粘贴链接 -> 选择目录 -> 导入。
- 只做与导入出装直接相关的功能，不扩展到战绩、自动更新、客户端操作等范围。
- 修改代码后必须运行对应测试；修改安装逻辑后必须用临时目录验证写入结果。
